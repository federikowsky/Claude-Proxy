<action>
Create `claude_proxy/infrastructure/providers/anthropic.py` following the exact Provider = Translator + Normalizer + Provider pattern from `openrouter.py`.

**Module structure (top to bottom):**

1. **Header:** `from __future__ import annotations`

2. **Imports (grouped: stdlib → third-party → local):**
   - stdlib: `logging`, `collections.abc.AsyncIterator`, `collections.abc.Mapping`, `typing.Any`
   - third-party: `httpx`
   - local domain: `ProviderAuthError`, `ProviderBoundaryError`, `ProviderHttpError`, `ProviderProtocolError`, `UpstreamTimeoutError` from `errors`; all event types + `ChatRequest`, `ChatResponse`, `ContentBlock`, `ModelInfo`, `ProviderRequestContext` from `models`; serialization functions from `serialization`
   - local infra: `ProviderSettings` from `config`; `SharedAsyncClientManager` from `http`; `IncrementalSseParser`, `SseMessage` from `sse`
   - `json_loads` from `claude_proxy.jsonutil`

3. **Module logger:** `_logger = logging.getLogger("claude_proxy.anthropic")`

4. **`AnthropicTranslator` class:**
   - `def to_payload(self, request: ChatRequest, model: ModelInfo) -> dict[str, object]:` — near-identical to `OpenRouterTranslator.to_payload()`: builds JSON body with `model`, `messages`, `max_tokens`, `stream`, plus optional `system`, `metadata`, `temperature`, `top_p`, `stop_sequences`, `tools`, `tool_choice`, `thinking`, `extensions`. Include the same `ProviderBoundaryError` check for invalid tool schemas.
   - `def to_count_tokens_payload(self, request: ChatRequest, model: ModelInfo) -> dict[str, object]:` — builds a subset for Anthropic's native count_tokens endpoint. Include: `model`, `messages`, `system`, `tools`, `tool_choice`, `thinking`. Exclude: `stream`, `max_tokens`, `temperature`, `top_p`, `stop_sequences`, `metadata`, `extensions`. The `model` field uses `model.name` directly.

5. **`AnthropicStreamNormalizer` class (per D-02 — forked normalizer):**
   - Instance state: `self._blocks: dict[int, ContentBlock] = {}` and `self._stream_finished: bool = False`
   - `def normalize(self, message: SseMessage) -> CanonicalEvent | None:` — **no `[DONE]` sentinel handling** (per D-03). Parse `message.data` as JSON, dispatch on `event_type`:
     - `message_start` → `MessageStartEvent` (extract `message` sub-object, build `ChatResponse` with `response_from_payload` style inline logic matching OpenRouter normalizer's `_message_start`)
     - `content_block_start` → `ContentBlockStartEvent` (use `content_block_from_payload(strict=False)`, store block in `self._blocks[index]`)
     - `content_block_delta` → `ContentBlockDeltaEvent` (use `delta_from_payload()` with stored block)
     - `content_block_stop` → `ContentBlockStopEvent` (pop from `self._blocks`)
     - `message_delta` → `MessageDeltaEvent` (extract `stop_reason`, `stop_sequence`, `usage`)
     - `message_stop` → `MessageStopEvent` (via `_emit_message_stop_once()`)
     - `ping` → `PingEvent`
     - `error` → `ErrorEvent`
     - unknown → `ProviderWarningEvent`
   - Private helpers: `_message_start()`, `_content_block_start()`, `_content_block_delta()`, `_content_block_stop()`, `_emit_message_stop_once()` — same pattern as OpenRouter normalizer.

6. **`AnthropicProvider` class:**
   - Constructor: `__init__(self, *, settings: ProviderSettings, client_manager: SharedAsyncClientManager, translator: AnthropicTranslator | None = None, parser: IncrementalSseParser | None = None)`
   - `async def stream(self, request, model, provider_context=None) -> AsyncIterator[CanonicalEvent]:` — same pattern as `OpenRouterProvider.stream()`. Use `AnthropicStreamNormalizer`. POST to `self._messages_url()`. Headers via `self._headers(accept="text/event-stream", provider_context=provider_context)`. No `params` method needed (Anthropic doesn't use query params).
   - `async def complete(self, request, model, provider_context=None) -> ChatResponse:` — POST to `self._messages_url()`, accept JSON, parse with `response_from_payload()`. Error mapping via `_raise_anthropic_http_error()`.
   - `async def count_tokens(self, request, model, provider_context=None) -> int:` — POST to `self._count_tokens_url()` using `translator.to_count_tokens_payload()`. Parse response JSON for `input_tokens` key. Return the integer value.
   - `async def _open_stream(self, stream_context) -> httpx.Response:` — same pattern as OpenRouter: enter context, check status, raise on error.
   - `def _messages_url(self) -> str:` — `f"{self._settings.base_url.rstrip('/')}/messages"`
   - `def _count_tokens_url(self) -> str:` — `f"{self._settings.base_url.rstrip('/')}/messages/count_tokens"`
   - `def _headers(self, *, accept: str, provider_context: ProviderRequestContext | None = None) -> dict[str, str]:` — returns dict with `x-api-key` (raw key, NOT Bearer), `anthropic-version` from `self._settings.anthropic_version`, `Accept`, `Content-Type: application/json`. If `self._settings.anthropic_beta` is not None, include `anthropic-beta` header. If `provider_context` has headers, merge them.
   - `def _timeout(self) -> httpx.Timeout:` — same as OpenRouter.

7. **Module-level helpers (copied from openrouter.py — per RESEARCH.md §9 recommendation):**
   - `_mapping(value)`, `_int_or_default(value, default)`, `_string_or_none(value)`, `_extras(payload, excluded)`, `_role_or_default(value)`, `_provider_error_message(body: bytes) -> str | None`
   - `_raise_anthropic_http_error(status: int, body: bytes) -> None:` — if status in `{401, 403}`: raise `ProviderAuthError` with message and `details={"provider": "anthropic", "upstream_status": status}`. Otherwise raise `ProviderHttpError` with `provider="anthropic"`.

**Key differences from OpenRouter adapter:**
- No `[DONE]` sentinel in normalizer (D-03)
- `x-api-key` header instead of `Authorization: Bearer`
- `anthropic-version` mandatory header from settings
- Optional `anthropic-beta` header from settings
- No `_params()` method (no query params)
- No `X-OpenRouter-Title` or `HTTP-Referer` headers
- Dedicated `_count_tokens_url()` method for native endpoint
- `to_count_tokens_payload()` builds purpose-specific subset (not a probe hack)

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Proxy → Anthropic API | Untrusted upstream: HTTP responses, SSE events, error payloads from api.anthropic.com |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | S (Spoofing) | API key in _headers() | mitigate | Key read from SecretStr via get_secret_value(), never logged. ProviderSettings.api_key has repr=False, exclude=True. |
| T-01-02 | T (Tampering) | SSE event payloads | mitigate | All JSON parsed with json_loads + type-checked before domain mapping. Invalid payloads raise ProviderProtocolError. |
| T-01-03 | I (Info Disclosure) | Error messages from upstream | mitigate | _provider_error_message() extracts only .error.message string. Raw body never exposed to client. |
| T-01-04 | D (DoS) | Malformed/infinite SSE stream | accept | IncrementalSseParser raises ProviderProtocolError on truncation. Read timeout (300s) limits hang time. Identical risk profile to existing OpenRouter adapter. |
</threat_model>

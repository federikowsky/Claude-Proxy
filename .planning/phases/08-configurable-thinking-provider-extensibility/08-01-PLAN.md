    **In `llm_proxy/infrastructure/config.py` — `ModelSettings`:**
    Add three new fields after `thinking_passthrough_mode`:
    ```python
    thinking_open_tag: str | None = "<think>"
    thinking_close_tag: str | None = "</think>"
    thinking_extraction_fields: tuple[str, ...] = ("reasoning_content", "reasoning")
    ```
    - `thinking_open_tag`/`thinking_close_tag`: The tag pair used to detect thinking content
      in streamed text. Set both to `None` to disable tag-based extraction entirely.
      Default matches current hardcoded `<think>`/`</think>`.
    - `thinking_extraction_fields`: Response delta field names checked for structured
      reasoning content (e.g. DeepSeek's `reasoning_content`). Default matches current
      hardcoded `("reasoning_content", "reasoning")`. Empty tuple disables field-based
      extraction.

    **In `llm_proxy/infrastructure/config.py` — `ProviderSettings`:**
    Add two new fields after `anthropic_beta`:
    ```python
    custom_headers: dict[str, str] = Field(default_factory=dict)
    finish_reason_map: dict[str, str] | None = None
    ```
    - `custom_headers`: Arbitrary HTTP headers merged into every request to this provider.
      Useful for provider-specific auth schemes, feature flags, tracing headers.
    - `finish_reason_map`: Override the default OpenAI `finish_reason` → Anthropic
      `stop_reason` mapping. `None` means use the built-in default map. When provided,
      this dict fully replaces the default (not merged). Keys are OpenAI finish_reasons,
      values are Anthropic stop_reasons.

    **In `llm_proxy/domain/models.py` — `ModelInfo` dataclass:**
    Add matching fields after `thinking_passthrough_mode`:
    ```python
    thinking_open_tag: str | None = "<think>"
    thinking_close_tag: str | None = "</think>"
    thinking_extraction_fields: tuple[str, ...] = ("reasoning_content", "reasoning")
    ```

    **In `llm_proxy/infrastructure/resolvers.py` — `StaticModelResolver.resolve()`:**
    Add the three new field mappings in the `ModelInfo(...)` constructor call:
    ```python
    thinking_open_tag=config.thinking_open_tag,
    ```

    (and vice versa). Raise `ValueError` if mismatched. This prevents broken tag pairs.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| YAML config → Settings | Config file loaded at startup, operator-trusted |
| ProviderSettings.custom_headers → upstream HTTP | Headers injected into upstream requests |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-08-01 | T (Tampering) | custom_headers | accept | Operator controls config file; no user input reaches custom_headers. Operator is trusted. |
| T-08-02 | I (Info Disclosure) | custom_headers | mitigate | custom_headers not logged at info level; debug-level logging already controlled by `debug: true`. No secret leak at default log level. |
| T-08-03 | S (Spoofing) | custom_headers | mitigate | custom_headers cannot override Authorization header — provider `_headers()` method will set Authorization AFTER merging custom_headers (Plan 02 ensures this ordering). |
</threat_model>
